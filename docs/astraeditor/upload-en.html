Submit extension for AstraEditor

Please Remember!

AstraEditor Extensions Gallery Dosen't need "Miscellaneous Tools" and other extensions that overlap with the "Extensions Gallery" feature!

You can import your extension directly without submitting it to the Extensions Gallery.

Remember

We use "Issues" to make it easier for you to submit extensions. This requires you to have a GitHub account before submitting an extension. For how to register, please search online; we won't go into detail here.

Preparation

Review

Before submitting an extension, make sure your extension can run correctly. We will review the extension you submit, including but not limited to:

  • The extension can load correctly
  • All added blocks work correctly
  • Does not contain discrimination, filthy language, internet memes or similar content
  • No excessive AI involvement (100% AI-generated)
  • You fully understand the extension you wrote (every line of code)

The review timeline is not fixed, but most reviews are completed within a week. Please check the results in time.

Packaging

You need to ensure that the files you submit are understandable to others. We have specified a simple directory structure to make it easier for both of us to check and add your extension:

ExtensionName.zip
    |- main.js      # Your extension file
    |- featured.png # Extension header image, displayed in the Extensions Gallery (you can also upload an .svg image)
    |- text.json    # Stores your extension information
Make sure file names are correct

If your submission does not meet our requirements, we may reject your extension.

Below we will explain the specific specifications for each file.

main.js

This is where your main extension file is stored. For how to write the extension, please refer to other extension tutorials.

That said, the beginning of your extension should contain the following:

Copy it! 
// Name: Extension Name
// ID: Extension ID
// Description: Extension Description
// By: Author 
// By: More Authors  (optional, more can be added)
// License: License (MIT, GPLv3, etc.)

This is mandatory in this Extensions Gallery, as it affects the actual display.

We require you to write Name and Description in English, which will be used as the default text displayed in the Extensions Gallery.

Note: For multiple authors, use multiple // By: lines instead of commas

Additionally, we do not require your content to include internationalization.

featured.png(.svg)

This is the extension's header image, which is the image displayed in the Extensions Gallery. We strictly require the extension image to meet the following conditions:

  • Aspect ratio of 2:1
  • Resolution no less than 600x300 (if it's a png)
  • Has a clear representation, whether text or graphic (you can even just write the extension name on it)

We also suggest that your extension image:

  • Not be monotonous
  • Be aesthetically pleasing (refer to TurboWarp and other Extensions Gallery extensions)
  • Have a clear and distinct theme color, rather than gaudy, unconventional gradients and AI-generated images

text.json

You can copy this example and modify it

Copy it! 
{
    "Name":{
        "zh-cn":"Extension Name"
        ...more languages
    },
    "Description":{
        "zh-cn":"Extension Description"
        ...more languages
    }
}
There is no need to add English here; English is added in the Name and Description fields within main.js

We strictly require your extension format to include the above two items, and Name and Description must include zh-cn. This is very helpful for internationalization.

We do not require your extension content to include internationalization.

Preparation

Issues

As mentioned above, we use Issues as the submission medium. We will explain the submission steps.

1. Go to the Extensions Repository — Issues

Go to this link: https://github.com/AstraEditor/extensions/issues

2. Create an Issue

Click New issue. In the input box below Add a title, enter something like Add new extension named "Extension Name" so we can identify it. In the Add a description field, enter any additional information you want to tell us; this is not required. Click the button that says Paste, drop, or click to add files and upload your prepared extension file (.zip). Wait for the upload to complete, then click Create!

Done! Now just wait for us to review and respond to you!
